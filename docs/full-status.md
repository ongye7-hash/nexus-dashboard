## 3. 알려진 버그나 문제점

### 확인된 문제
| 문제 | 심각도 | 설명 |
|---|---|---|
| 터미널 서버 미테스트 | 중 | xterm.js + node-pty 조합이 실제로 동작하는지 Windows에서 미확인 |
| VPS SSH 미테스트 | 중 | 실제 VPS에 SSH 연결 테스트 안 해봄 |
| Claude Code 파이프 | 중 | `cat review.md \| claude` 명령이 실제로 작동하는지 미확인 |
| 프로세스 이름 감지 | 낮 | wmic + HTTP title 방식이 모든 케이스를 커버하지 못할 수 있음 |
| Turbopack 경고 | 낮 | 빌드 시 lockfile 경고 (기능에 영향 없음) |
| 히트맵 데이터 비어있음 | 낮 | 사용 시작 전이라 활동 데이터가 축적되지 않음 |

### 구조적 한계
| 한계 | 설명 |
|---|---|
| Windows 전용 | 경로/명령어(explorer, wt, netstat)가 Windows 종속 |
| Desktop 하드코딩 | 스캔 경로가 C:\Users\user\Desktop 고정 |
| 단일 사용자 | 인증/멀티유저 없음 |
| 테스트 없음 | 자동화된 테스트 파일 0개 |

## 6. 매일 쓸 때 불편할 수 있는 점

| 불편 | 이유 | 해결 방향 |
|---|---|---|
| **아침에 브라우저를 열어야 함** | 바탕화면에 바로가기 있어도 한 단계 거침 | Electron 앱 또는 PWA 자동 시작 |
| **설정이 분산됨** | GitHub은 사이드바 설정, VPS는 VPS 설정, AI는 모달 안 | 통합 설정 페이지 |
| **히트맵/스트릭이 처음엔 비어있음** | 데이터 쌓이기 전까지 의미 없음 | 기존 git log에서 과거 데이터 import |
| **터미널 폰트/크기 조절 안 됨** | xterm.js 설정이 하드코딩 | 터미널 설정 옵션 추가 |
| **스캔 경로 고정** | Desktop 이외 위치의 프로젝트 못 봄 | 스캔 경로 추가 기능 |
| **프로젝트 많으면 로딩 느림** | 모든 프로젝트의 git remote를 execSync로 확인 | 캐시 또는 비동기 처리 |
| **모바일에서 터미널 사용 어려움** | 가상 키보드로 터미널 입력이 불편 | 모바일용 Quick Action 버튼 |

## 7. 다른 프로젝트/서비스와 연결 시 시너지

### VPS + Nexus
| 시너지 | 설명 |
|---|---|
| **배포 원클릭** | 로컬에서 git push → VPS에서 자동 pull + restart |
| **로그 모니터링** | VPS 프로젝트의 pm2 logs를 대시보드 터미널에서 실시간 확인 |
| **환경 변수 동기화** | 로컬 .env.example → VPS .env 자동 비교 |

### GitHub + Nexus
| 시너지 | 설명 |
|---|---|
| **PR 상태 표시** | 프로젝트 카드에 "열린 PR 2개" 뱃지 |
| **Issue 연동** | GitHub Issue를 프로젝트 TODO로 가져오기 |
| **Actions 상태** | CI/CD 빌드 성공/실패를 대시보드에 표시 |
| **릴리스 관리** | GitHub Release를 대시보드에서 생성/확인 |

### AI + 나머지 전부
| 시너지 | 설명 |
|---|---|
| **VPS 장애 분석** | 서버 에러 로그를 Claude에게 분석 요청 |

### SaaS 확장 시
