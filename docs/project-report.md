# Nexus Dashboard 프로젝트 상세 보고서

---

## 1. 프로젝트 정체성

**한 줄 요약:** Windows 바탕화면의 모든 개발 프로젝트를 한 화면에서 모니터링하고 조작하는 로컬 개인 대시보드

**기술 스택:** Next.js 16 + React 19 + Tailwind CSS 4 + SQLite + Ollama(AI)

**총 코드량:** 약 12,000줄 (컴포넌트 7,190줄 / API 2,532줄 / DB+유틸 892줄 / 훅 575줄 / 메인페이지 881줄)

**컴포넌트:** 25개 / **API 라우트:** 17개 / **DB 테이블:** 8개 / **DB 함수:** 40개+

## 3. 기술 아키텍처

```
브라우저 (React 19 + Tailwind CSS 4)
    |
    +-- 커맨드 팔레트 (cmdk)
    +-- 애니메이션 (Framer Motion)
    +-- 마크다운 렌더링 (react-markdown)
    +-- 아이콘 (Lucide React)
    |
    v
Next.js 16 API 라우트 (17개 엔드포인트)
    |
    +-- SQLite (better-sqlite3) --- 8개 테이블, 40개+ 함수
    +-- Git CLI (execSync) --- 커밋/푸시/풀/상태/로그
    +-- 파일시스템 (fs) --- 프로젝트 스캔/파일 트리/README
    +-- 프로세스 관리 (netstat/tasklist) --- 서버 감지/종료
    +-- Ollama HTTP (localhost:11434) --- AI 기능
    +-- ripgrep/findstr --- 코드 검색
    +-- chokidar --- 실시간 파일 감시 (SSE)
```

## 4. 어디에 쓰이면 좋을지

### 가장 적합한 대상

**"프로젝트가 5개 이상인 1인 개발자"**

- 프리랜서가 동시에 여러 클라이언트 프로젝트를 관리할 때
- 사이드 프로젝트를 여러 개 돌리는 개발자
- 부트캠프/학원 수강생이 과제 프로젝트를 관리할 때
- 풀스택 개발자가 프론트/백/인프라 여러 repo를 동시에 볼 때

### 확장 가능한 방향

| 방향 | 설명 | 난이도 |
|---|---|---|
| 팀용 | 공유 서버에 올려서 팀원들 프로젝트 현황판으로 사용 | 중 |
| 교육용 | 코딩 부트캠프에서 학생들 진행상황 모니터링 | 중 |
| 포트폴리오 | 스트릭/뱃지/히트맵 데이터를 외부 공개 페이지로 내보내기 | 하 |
| CI/CD 연동 | GitHub Actions/Vercel 배포 상태를 대시보드에 표시 | 중 |
| Mac/Linux 지원 | 경로 로직만 바꾸면 크로스 플랫폼 가능 | 하 |
| Electron 앱 | 시스템 트레이 상주 + 알림 기능 추가 | 중상 |
| SaaS화 | 클라우드 버전으로 만들어서 GitHub 연동 대시보드 서비스 | 상 |

### 안 맞는 곳

- **대규모 팀** -- Jira, Linear 같은 이슈 트래커가 더 적합
- **CI/CD 파이프라인 관리** -- GitHub Actions, Jenkins의 영역
- **코드 리뷰** -- GitHub PR이 더 적합
- **프로젝트가 1~2개인 사람** -- 이 도구가 필요 없음

## 5. 현재 한계

| 한계 | 이유 |
|---|---|
| Windows 전용 | 경로, 명령어(explorer, wt, netstat)가 Windows 종속 |
| Desktop 폴더 고정 | 스캔 경로가 C:\Users\user\Desktop 하드코딩 |
| Ollama 필수 | AI 기능은 로컬에 Ollama 설치+실행 필요 |
| 테스트 0개 | 자동화된 테스트 없음 |
| 인증 없음 | localhost 전용이라 인증 불필요하지만, 외부 노출 시 위험 |
| 단일 사용자 | 멀티유저/팀 기능 없음 |

## 7. 이 프로젝트를 쓰는 이유

**내 모든 프로젝트의 상태를 한눈에 보는 것.**

- 어떤 프로젝트에 커밋 안 한 변경사항이 쌓여있는지
- 지난주에 어떤 프로젝트에 시간을 제일 많이 썼는지
- 3주째 안 건드린 프로젝트가 뭔지

이걸 각 폴더 열어보기 전에 알려주는 도구.

## 8. 결론

이 프로젝트는 **"많은 프로젝트를 혼자 관리하는 개발자의 개인 관제탑"** 이다. 기능은 이미 충분히 많고, 코드도 12,000줄로 소규모 프로덕트 수준이다.

**가장 가치 있는 활용:**
1. 본인이 직접 매일 쓰면서 데이터를 쌓는 것
2. 포트폴리오 프로젝트로 제출하는 것 (기능 다양성, 기술 스택 최신성)
3. 오픈소스로 공개해서 비슷한 니즈를 가진 개발자들에게 배포하는 것
